# Algomagia（v2）

計算理論を魔法にする**ノベルRPG**（RPG主体・物語多め）。テーマ＝**決定論SQ vs 自由意志exSQ**、魔石＝キュビット。
Web優先（Phaser 4 + TypeScript + Vite）。**旧プロジェクト `../20260608_algomagia` は遊べるアーカイブ**（参照可・改変しない）。
再開時はまず @HANDOVER.md（入口）→ 次の作業は @WORKPLAN.md。基盤決定は @docs/adr/0001-v2-foundation.md、世界観は @docs/design/world-bible.md。

## コマンド
- `npm run dev` — 開発サーバ(Vite)
- `npm run test` — Vitest（変更後は必ず実行）/ `npm run test:watch`
- `npm run typecheck` — tsc --noEmit（変更後は必ず実行）
- `npm run build` — typecheck込みの本番ビルド

## アーキテクチャ
- `src/core/` — **決定論的なゲームロジック。UIに依存させない。**ここを単体テストで固める。**v1からは移植せず二軸モデルで新規実装。**
- `src/app/` — Phaser 層（起動/シーン/描画/テーマ）。core を呼ぶだけにする。
- `src/game/` — シーンとデータ（着手はこれから）。
- `docs/` — adr(意思決定) / design(安定設計=世界観など) / plans(実装待ち仕様) / process(開発運用)。

## 基盤の不変条件（ADR-0001 v2）
- **二軸**: キャラ＝**自由意志**（最大/現在＝燃料・耐久）／魔石（装備）＝**演算・心域・霊脈**＋積めるスキル。スキル＝自由意志×魔石。
- **テーマ＝設計契約**: 最適化は「**出力不変・コスト減**」を**設計契約**として守る（実計算量カウントはしない＝ADR-0001 D3）。
  最適化＝より安い手書きバリアントへの差し替え。進行ゲート（あかずの間）は cost ≤ X を最適化で満たして開く。
- **残す概念**: スキル習得・解析・最適化・**看破（予測で防御）**。**魔素＝実ops は撤廃**（旧 ADR-0002 は LEGACY）。
- **乱数は必ずシード付き**（素の `Math.random()`/`Date.now()` 禁止）。決定論を保つ。

## 規約
- core は副作用なし・純関数寄り。新ロジックは必ずテストを添える（決定論・UI非依存）。
- 用語: 自由意志=スキルの燃料(有限)／魔石=ハード(演算/心域/霊脈)／看破=予測防御／最適化=出力不変のコスト減。
- ゲージ間（演算/心域/霊脈）で数値を直接比較しない（別単位）。
- TS編集後は hooks が自動 typecheck。テストは `npm run test` を自分で回す。

## ワークフロー
- 大きめの変更は計画→実装（core を先にテスト付きで）→検証（typecheck/test）→ドキュメント同期（design/adr/plans の役割で置き場所を決める）。
- 設計は方式A（小さい複数案を出して ponti が選ぶ）で進める。世界観の更新は world-bible.md に反映。
