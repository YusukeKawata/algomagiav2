# 魔石盤（回路パズル）と工房 ——魔法システムの中核

> 作成: 2026-06-19 / ステータス: 設計確定（骨子）＋未決の数値・細部あり。
> 基盤決定 [ADR-0001](../adr/0001-v2-foundation.md)。世界観 [world-bible.md](world-bible.md) §8.5/§8.6/§8.9/§8.12/§8.13 と噛み合わせる。
> ここは ponti と方式A（小さい複数案→選ぶ）で共同設計した中核機構。**実装はこれから**（最小核から、後述）。

## 1. 一言で

魔法システム＝**回路パズル**。魔石を**2D盤**に駒のように嵌め、**自由意志を左から流す**と魔石の**光る文様（配線）**が光り、
**左の入口→右の出口まで一本につながった経路＝スキル1発**になる。つながった経路の数だけ戦闘で撃てる。
**回路＝計算、自由意志＝電流、最適化＝盤上の配線取り回し**——「これは魔法じゃない、計算だ」が手触りになる。

## 2. 盤（主人公のステータスで広がる）

- **横幅＝心域**、**縦高＝演算**（どちらも**主人公のステータス**。成長/覚醒で増える）。
  - 心域 1→2 で横に1マス増、演算 1→2 で縦に1マス増。
  - **心域が一定値を超えると盤が2枚目・3枚目**と増える。
- **入口＝左端、出口＝右端**。自由意志は左から流れる。
- 三ゲージの意味（[ADR-0001 D4 改訂](../adr/0001-v2-foundation.md)）：
  - **演算＝縦＝並列レーン数＝並列度**（同時に成立できる経路＝同時に撃てるスキル数。§8.13 並列演算と一致）。
  - **心域＝横＝経路の深さ／盤の枚数＝保持容量**（長い/多い回路を保てる。§8.9 回復＝心域と一致）。
  - **霊脈＝精霊魔法用の別レーン／別盤（外部接続）**（§8.12。自前計算でなく"借りる力"なので盤の本線とは別系統）。

## 3. 魔石（駒）

- 各魔石（キュビット）は盤に置く駒。持つもの：
  - **文様（配線）**：どの辺どうしをつなぐか（直線/曲がり/分岐…＝回路タイル）。
  - **属性**（炎/氷/…等）と **数値（ステータス）**。
- **回収＝解析**：モンスターを倒すと、その魔石が走らせていた計算を読み解いて駒として手に入る（旧「敵の式を学ぶ」）。

## 4. 発動とスキルの決まり方（確定）

- 自由意志を左から流す → 文様が光る → **入口から出口まできれいに一本つながった経路＝1スキル発動**。
- **つながった経路の数＝戦闘で撃てるスキルの数**（＝並列度＝演算＝縦レーン数）。
- **発動するスキルは、その経路上の魔石の数値の合計で決まる**（強さ等）。「核石」は無し＝経路全体の集計。

## 5. 三操作（工房）の対応

- **解析**：倒した魔物の魔石を読み、**駒（魔石）を手に入れる**（インベントリに追加）。
- **集積**：駒を**盤に嵌めて回路を組む**こと。＋安い魔石を素材に消費して装備魔石を育てる（§7 経済のシンク）。
- **最適化**：**盤上の配置を組み替えて、同じ盤・同じ自由意志でより多く/強い経路を通す**こと＝この空間パズルそのもの。
  「**出力不変・コスト減**」が空間に翻訳される（ADR-0001 D3 の設計契約）。
  - **進行ゲート**：あかずの間（§8.11）は「cost ≤ X の経路 or 一定の出力」を要求＝盤を最適化して満たすと開く。

## 6. 最適化の魂（ADR-0001 D3 との接続）

実 ops は数えない。代わりに**盤の配線という見える形で最適化を体感**させる：少ない自由意志で、より長い/多い/強い回路を通す配置を見つける＝アルゴリズムを良くする快感。
古典的な「後で能力を得て戻る」ゲートを、**盤の最適化＝鍵**として実装する。

## 7. 魔石の経済（階層＋一生）〔確定〕

1トークンが 通貨/素材/装備/Q破片 を兼ねる衝突を、**階層＋ライフサイクル**で解く：
- **生キュビット**（雑魚ドロップ・量産）＝**通貨**・**集積の素材**として消費。
- **芯**（大魔物/遺構の希少石）＝**盤に装備して育てる本体**。
- 育て切った先＝**Q破片**。**集積が消費シンク**、**Q再建が長期目標**＝伏線（§8・§10）と地続き。

## 8. 追加で確定（2026-06-19・推奨採用＝暫定・改訂容易）

- **駒の形＝配線優先**：基本は**1マスの回路タイル（辺コネクタ）**。深さは"文様をつないで左→右に通す"こと。
  希少/大魔石のみ後で複数マス形へ拡張（"2Dパズルではめる"感覚は上位層で）。**最小核は1マスのみ。**
- **スキルの"種類"＝経路の属性構成で決まる**：経路上の魔石の**属性の組成**でスキルの属性/種類が、**数値の合計**で強さが決まる。
  同じ盤でも、どの石を一本につなぐかで別のスキルが出る＝回路を組むこと自体が魔法を作る。**最小核は1属性・強さのみで先行。**

### 実装メモ（第1幕・2026-06-21／ponti 指示で確定）
- **文様(edges)は変更不可**：魔石はモンスターから**ランダムでドロップ**し、プレイヤーは**手持ちをやりくり**して盤に嵌める（思い通りの文様が欲しければ何度も戦う）。盤に嵌めるときに属性を選ぶことはしない。
  - 〔改訂・2026-06-21 続〕**value/属性は第2幕の工房（集積）で変えられる**（ponti が決定を緩和）：`fuseValue`＝素材を捧げて魔素量+1／`fuseAttr`＝素材の属性へ付け替え（弱点を意図的に狙える）。**文様(edges)だけは依然不変**。地中の里の石工リーゼで解禁（`WorkshopScene`）。＝「ドロップをやりくり」の芯は残しつつ、集積に育成と弱点設計の手応えを与える。
- **最初のスロットは 1×1**：覚醒直後の盤は心域1×演算1。**ガロが最初に手渡す魔石＝「─（左右一本線）・物理属性・魔素量1」**＝1×1 にちょうど1本の回路が成立する入門。
- **魔石のステータス表示＝「魔素量」**（＝データモデルの `value`／強さ寄与。語は ponti 指定。ADR-0001 で退けた「魔素＝実ops」とは無関係の表示名）。
- **編集は戦闘の外**：盤の配置はメニュー（戦闘外）で行い、戦闘では組んだ回路を「スキル」コマンドで撃つだけ＝「はめる画面に戦闘は不要」を満たす。
- **盤は戦って育つ——ただし“解禁済みゲージの方向だけ”（2026-06-22 修正・ponti 指摘）**：成長は world-bible の**解禁順**に従う＝**自由意志（覚醒）→ 心域＝横（第二の里の工房）→ 霊脈 → 演算＝縦＝並列度（並列の町）**。
  - `state.nextBoardDims(mind,compute,cap,unlock)`＝**横(心域)は `unlock.mind`、縦(演算)は `unlock.compute` が真のときだけ伸びる**（心域優先）。
  - **覚醒直後＝心域も演算も未解禁＝盤は 1×1 のまま**。レベルが上がっても伸びない（＝「村を出てもスロットは1つ」）。`grantXp` の盤拡張は `mindUnlocked || computeUnlocked` のときだけ走る。
  - **第二の里で `unlockMind()`**：心域を解禁し上限を 4 へ。**到達レベルぶん横へ一気に追いつく**（`catchUpBoard`）＝解禁の瞬間に複数の横スロットが使える。以後レベルアップで横が伸びる。ただし**演算は未解禁なので縦は1のまま＝盤は最大 `4×1`＝成立回路は常に1本＝撃てるスキルは1つ**（並列度1。world-bible「演算＝並列は並列の町で」と一致）。
  - **並列の町で `unlockCompute()`**（第2幕後半・実装はこれから）：演算を解禁し縦が伸びる＝複数回路＝複数スキル同時（並列度）。
  - 〔旧仕様・撤廃〕「覚醒後レベルアップで心域=横→演算=縦へ無条件に広がる／覚醒時 `catchUpBoardToLevel` で一気に拡張」は world-bible の解禁順に反していた（心域・演算が覚醒時点で勝手に開いていた）。**解禁ゲートに置換**。
- **スキル火力カーブ（2026-06-21 続）**：スキルダメージ＝**強さ合計＋長回路ボーナス（超線形・+25%/石）**＝`core/combat.skillBaseDamage`。長い回路ほど効率がよい（回路＝計算が深いほど強力）＋弱点×2＝**育った盤の長い弱点回路がこうげきを上回る**。コストは石数（線形）。1×1の単石は弱点でも2＝序盤はこうげき優位＝softlock維持（`autoWinnable` は攻撃のみで判定）。
- **心域＝回復魔法＝“回復属性石の回路”（2026-06-22 改訂・ponti 指示で回路化）**：§8.9「損傷前の状態を心域から読み戻す」を、専用コマンドでなく**盤の回路**で表す＝§61「スキルの種類は属性組成で決まる」に一致。
  - 地中の里で石工リーゼが**回復属性石（`RIESE_STONE`＝─・heal・魔素量6）**を手渡し、同時に **`unlockMind()`** で心域＝横を解禁する。
  - 盤に嵌めて heal-優勢の回路を組むと、その回路は戦闘の「スキル」で撃つと**HP回復**になる（`core/board` に `Attr 'heal'`／`core/combat.heroSkill` が `element==='heal'` を回復に分岐）。回復量＝`skillBaseDamage`（＝強さ合計＋長回路ボーナス。長い回復回路ほど大きく戻せる）。コスト＝石数の自由意志。
  - 〔撤廃〕旧・専用コマンド「いやし」（`heroMend`/`MEND_COST`/`mendPower`/flag `healMagic`）。演算ロック中は盤が最大 `4×1`＝回路は常に1本なので、回復回路を組むと攻撃回路は組めない＝**こうげき(無料)で攻め、単一スキル枠を回復に使う**トレードオフになる（並列＝演算は並列の町で解禁）。

### データモデル（最小核・実装の正）
- `Edge = 'L' | 'R' | 'U' | 'D'`（駒の4辺）。
- `Stone = { id; edges: Edge[]; value: number }` ＝ 1マス駒。`edges` は**互いに導通する辺の集合**（例 `['L','R']`＝直線、`['L','R','U','D']`＝十字、`['U','D']`＝縦線）。`value`＝強さ寄与。属性は最小核では省略（後で `attr` 追加）。
- `Board = { width; height; cells }`（width=心域, height=演算）。`cells[y][x] = Stone | null`。
- **隣接導通**：(x,y) と (x+1,y) は前者に `R`・後者に `L` があれば導通／(x,y) と (x,y+1) は前者に `D`・後者に `U` があれば導通。
- **入口/出口**：左端列(x=0)で `L` を持つ駒＝入口レール接続、右端列(x=width-1)で `R` を持つ駒＝出口レール接続。
- **回路＝スキル**：入口レールと出口レールの両方に届く連結成分が1つ＝スキル1発。**強さ＝その成分に含まれる魔石の value 合計**。成立した回路の数＝撃てるスキル数。

## 9b. これから詰める未決（方式A）

- **自由意志のコスト**：発動ごとに現在値を消費（§5 代償）。1経路あたりの消費量の決め方。
- **属性の組成→スキル種類**の対応表（最小核の次）。
- **看破/防御**を盤でどう表すか（§8.5 予測防御との接続）。
- **霊脈の別盤/別レーン**の具体。駒の複数マス形・複数盤。数値設計全般。

## 9. 最小プレイ核（実装の入口）〔スコープ管理〕

いきなり全部は大きい。**最小核**を先に動かしてから拡張する：
1. **1枚盤・1マス駒（辺コネクタのみ）・強さのみの石・1属性**。左→右が一本つながったら1スキル発動、強さ＝経路数値合計。
2. → 属性、複数経路（並列＝縦）、盤の成長（心域/演算）。
3. → テトリス形駒、複数盤、霊脈の別盤、集積の消費アップグレード。

この順なら、最初から「回路を組む→戦闘で効く」コアループを早く触れる。
