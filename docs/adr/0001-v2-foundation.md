# ADR-0001（v2）: 新生 Algomagia の基盤決定

> 作成: 2026-06-19 / ステータス: 採用
> 旧プロジェクト `20260608_algomagia` から世界観 v2 を新規プロジェクトとして作り直すにあたっての基盤決定。
> 旧プロジェクトは **遊べるアーカイブとして温存**（消さない・参照可）。

## 背景

旧実装（v1）は「計算理論を魔法にするJRPG」を Phaser 4 で完成させた（タイトル→3章→ラスボス）。
だが共同設計してきた**世界観 v2（決定論SQ vs 自由意志exSQ・魔石=キュビット・魔石を装備する二軸成長）**が
v1 実装から大きくズレ、v1 コードに後付けすると設計と喧嘩する段階に達した。
そこで**新フォルダで殻（UI/物語/コンテンツ）をゼロから作り直す**。世界観の正典は [world-bible.md](../design/world-bible.md)。

## 決定

### D1. ジャンル＝ノベルRPG（**RPG主体**・テキスト多め）
プレイヤーが過ごす中心は RPG（探索・戦闘・成長）。物語/世界観を随所で明かすため文量は多いが、VN ではない。

### D2. 技術＝Phaser 4 + Vite + TypeScript + Vitest（v1 と同じスタックを継続）
RPG主体（地図・フィールド・戦闘演出）に canvas エンジンが自然。エージェント（Claude）が全工程をコード/テストで駆動でき、Web 配布が堅い。
- **不採用: Ren'Py** — VN エンジンで RPG は接ぎ木になる／Web 書き出しがβで不安定／TS資産を Python へ書き直す損失。
- **不採用: RPG Maker MZ** — コンテンツが GUI エディタと独自データに縛られエージェント開発に不向き・カスタム計算機構と衝突。
- **不採用: Phaser を捨てて DOM 中心** — 地図/フィールド/戦闘演出に結局 canvas が要る。

### D3. テーマの厳密さ＝**中間（設計契約）**。実計算量(ops)→コストの縛りを撤廃
v1 は素演算回数を実カウントしコスト（魔素）に直結させた（[ADR-0002 LEGACY](0002-determinism-and-mana-LEGACY.md)）。これが**ゲーム開発の自由度を強く縛っていた**ため撤廃する。
- **残す概念**: スキルの習得・**解析**・**最適化**・**看破**（予測で防御）。
- **新しいコストの決め方**: スキルの自由意志コストは**手書きのコスト階層**で与える。
- **最適化＝そのスキルをより安い手書きバリアントへ差し替える**こと。「**出力不変・コスト減**」は
  **実行時チェックではなく設計契約**として守る（プレイヤーは"アルゴリズムが安くなった"を体感する）。
- **進行ゲートは維持**: 例「あかずの間」は cost ≤ X を要求し、最適化で X 以下に下げて開く。
- → ops カウントのインタープリタは捨てるが、テーマの魂（決定論／最適化／予測）は残る。

### D4. ステータスを二軸に分離（魔石=ハード / 自由意志=あなた）〔同日改訂〕
世界観 §8.6 の二軸モデルをデータ構造に落とす。
- **キャラクター**: **自由意志**（最大値/現在値＝燃料・耐久。成長で最大値増・スキルで消費・泉/休息で回復・酷使で永続摩耗）＋
  **演算・心域・霊脈**（＝**魔石盤の寸法**。演算＝縦＝並列レーン／心域＝横＝経路の深さ・盤枚数／霊脈＝精霊魔法用の別レーン）。
- **魔石（駒）**: 盤に置く駒。**文様（配線）・属性・数値**を持つ。倒した魔物から回収（解析）し工房で育てる。
- スキル＝自由意志（燃料）× 魔石盤の回路（増幅器）。**魔法システムの中核＝魔石盤（回路パズル）**は
  [magic-stone-workshop.md](../design/magic-stone-workshop.md) を正とする。
- 〔改訂理由・同日〕当初は「演算/心域/霊脈＝魔石の三機能」としたが、集積機構を**回路パズル盤**に決めた結果、
  これら三軸は**盤の寸法＝主人公ステータス**へ移し、魔石は固有属性/数値を持つ**駒**に再定義した（意味は二重保持で §8.9/8.13 と一致）。

### D5. salvage（救出）方針
新コードベースに、v1 から **content-free な土台のみ**移植した:
設定（package.json/tsconfig/vite.config/index.html/.gitignore）・hooks（PostToolUse typecheck）・Phaser 起動の最小骨組み。
v1 のコンテンツ（core 機構 M1〜M7・敵/マップ/戦闘経済・各シーン・scenario.test）は**持ち込まない**。必要になったら参照して作り直す。

## 帰結
- 旧 ADR-0002 は **LEGACY**（決定論というテーマは継承、魔素＝実計算量カウントは緩和）。
- `src/core` は v1 を移植せず、二軸モデルに合わせて新規実装する（決定論・UI非依存・Vitest で固める方針は継承）。
- 解像度/ピクセル戦略（pixelArt 320×180 拡大 vs ネイティブ）は **未確定**（土台は text-friendly な 1280×720 ネイティブで起動）。
- アート供給（立ち絵/背景）・規模/尺は **未決**（着手前に詰める）。
