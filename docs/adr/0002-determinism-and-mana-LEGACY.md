# ADR-0002: 決定論と魔素の定義（最重要の不変条件）

- 日付: 2026-06-09
- ステータス: 採用（このプロジェクトの背骨。破ると世界観もバランスも崩れる）

## 決定（不変条件）
1. **魔素 = 決定論的に数えた計算コスト**。実時間・実メモリ・実CPUは**絶対に測らない**
   （端末差で不公平になり再現不能になるため）。素演算(+,-,*,/ 等)の回数で定義する（`src/core/ops.ts`）。
2. **ダメージは値の大きさ(magnitude)で測らない**。項数・節数・距離・ビット数など**“個数”**で測る。
   → 「ただ大きくする」最大化戦法を無効化（基準M）。
3. **最適化は飾りの倍率にしない**。アルゴリズムを本当に書き換えて魔素を減らす。出力は不変、魔素のみ減少。
4. **乱数は必ずシード付き**（`Math.random()`/`Date.now()` を素のまま使わない）。同入力→同結果。
5. ゲージ(時間/空間/通信/回路/IO)は**別単位**。ゲージ間で数値を直接比較しない。
6. **【根幹】スキル＝関数の組み合わせ**。スキルは複数の関数構成ノードを組み合わせた結果として
   生まれる1つの関数 f(x) であり、各ノードは f の**構造（次数・係数・族）を実際に組み立てる**。
   「大きさ(magnitude)を作るだけ」のノードは攻撃に寄与しない（2項より magnitude は無関係）。
   最適化は f の**評価/構成を安くする**こと（Horner / 高速累乗 等、出力 f は不変）。
   → 種火=低次係数を作る / 増幅=次数を上げて高次係数を足す / 収束=f を評価、のように
   「組み合わせると本当に最終式 f になる」よう実装する。これは本作の出発点・根幹。
